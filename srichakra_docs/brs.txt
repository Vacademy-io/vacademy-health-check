Business Requirement Specification (BRS)
1. Applications Involved
• ERP System
o Centralized repository for Purchase Orders (POs), approvals, and financial
transactions.
• Supplier Web App
o Portal Views:
▪ Administrators (Web Portal)
▪ Procurement Field Team (Android App)
▪ Suppliers (Lightweight Web App via link)
• WhatsApp Integration
o Notifications (daily greetings, reminders, alerts)
o Enquiries (two-way communication with suppliers)
2. Multi-Lingual Support
• Supplier Web App: Interface available in multiple languages.
• WhatsApp Integration: Messages delivered in supplier’s opted language.
• Language Change Option: Supplier can switch preferred language anytime.
3. Supplier Web App Portal
i. Supplier Onboarding
• Consent Form
o Aadhaar-based OTP signature (template provided by Business).
• Declaration by Field Officer
• Preliminary GST Verification
• Phone & Email Verification
• Daily Greetings via WhatsApp (in opted language) with 3 options:
o Navigate to Portal: Opens supplier web interface.
o Let’s Meet: Creates task for Field Officer; supplier notified via App.
o Change Language: Updates preferred language.
ii. Supplier Evaluation (SE)
• Triggered automatically post-onboarding.
• Evaluation scope: Owner + Employees.
• Supplier Improvement Plan (SIP):
o Manual creation option.
o Automatic SIP generation based on SE responses.
iii. Supplier Web Portal Access
• Authentication: OTP-based login.
• Supplier Menu Options:
o Material Rate Card
o Share Quotes (rates + quantity)
o Transaction History
o Ledger
o QC Reports
o Debit Notes
o Record Shipment Details
o Raise Enquiry
iv. Admin View (Supplier Web App Portal)
• Daily Rate Card Updates
o Product Category, Material, Area-specific.
o Supplier-wise incentive prices.
o Manual input for PlasticWala Price.
• Supplier Audit Calendar
o Track NCs to closure with photo/video proofs.
• SIP Monitoring
o Track trainings to closure with photo/video proofs.
• Supplier Requests Tracking
o Reassign requests based on Field Officer availability.
v. Field Officer’s App Portal
• Supplier Onboarding (upload KYC documents).
• Respond to supplier quotes.
• Generate POs: Push to ERP for approval workflow.
• Track Shipments
• Address Enquiries and track to closure
• Escalation Mechanism for issues with Supplier
6. Key Compliance & Security Requirements
• Aadhaar OTP-based consent for onboarding.
• GST, Email, Phone, Address verification during onboarding.
• OTP authentication for portal access.
• Secure integration between ERP ↔ Supplier Web App ↔ WhatsApp.

The supplier portal (SSMP) will centralize supplier interactions, daily updates, and task management while respecting language preferences (09:50).

Daily WhatsApp Greetings and Navigation Options (17:50)

Language change option available to accommodate supplier preferences and improve engagement.
Task Management and Escalations (54:00)

Queries raised by suppliers create tasks assigned to field officers, with comments and status tracking until closure.
Free-text query option preferred over detailed category selection to simplify user experience and reduce complexity.
Internal escalation mechanism enables suppliers’ issues to be escalated to managers if field officers cannot resolve them.
Admin has authority to reassign tasks or suppliers to different field officers temporarily or permanently.
Supplier Query and Dispute Window (53:30)

Time-bound windows (e.g., one week) limit when suppliers can raise disputes on QC reports, debit notes, or payment advice.
After the predefined period, related inquiry options are removed from supplier portal menus.
General inquiries can be raised anytime, supporting ongoing communication without limitations.
Supplier Transaction History Access (52:30)

Suppliers can filter and fetch transaction details by PO number, invoice, or date range.

Pricing, Rate Cards, and Purchase Order Workflow

Dynamic Supplier-Specific Pricing Model (29:30)

Field officers generate PRs from supplier quotes directly within SSMP, which triggers creation of POs in ERP Next via APIs.
ERP Next handles PO approval workflows; only approved POs are communicated to suppliers.
PO details sent back to SSMP for record-keeping and supplier communication via WhatsApp or portal.
ERP Next also sends official PO emails to suppliers and field officers for audit trail and compliance.
